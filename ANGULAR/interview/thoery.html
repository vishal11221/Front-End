  8. What is standalone component?
    1. It is a feature of Angular that allows you to create components without declaring them in an NgModule. These components are marked as standalone: true and can be used directly in routing, other components, or features.
    2. A standalone component is not declared in any NgModule and can be directly used in routes, other standalone components, or modules without needing to be declared again.
  9. What are NgModule ?
    1. NgModule is a feature of Angular that allows you to group related components, directives, and other providers together.
  10. Difference between Component and Directive
    1. A Component is a type of directive with a template. Components are used to create views and handle user interactions.
    2. A Directive, on the other hand, does not have its own template. It is used to add behavior or modify the DOM of existing elements.
  11. Difference between Component and Module ?
    1. Component is a class that defines the structure and behavior of a UI element.For example login component, singup component, profile component
    2. Module is a container for related components, directives, and other providers. For example User Module inside login component, singup component, profile component
  12. Directives & There types
    1. In Angular, Directives are special markers on a DOM element (such as an attribute, element, or class) that tell Angular to perform specific actions or apply specific behavior to that element. Directives play a vital role in Angular, helping you to extend HTML capabilities and modify DOM behavior dynamically.
    2. There are three main types of directives in Angular:
      1. Component Directives (a special type of directive):-Components are essentially directives with their own templates. Every Angular component is a directive with a template.selector: The app-example selector can be used in HTML like to invoke this component.
      2. Structural Directives:-Structural directives alter the layout of the DOM by adding, removing, or manipulating elements. They change the structure of the view. These directives use the * symbol when applied in templates.
        1. *ngIf: Conditionally includes/excludes elements.
        2. *ngFor: Repeats an element for each item in a list.
        3. *ngSwitch: A set of directives to switch between different templates.
      3. Attribute Directives:-Attribute directives change the appearance or behavior of an element. Unlike structural directives, they do not modify the DOM structure but can manipulate DOM attributes.
        1. ngClass: Adds/removes a set of CSS classes.
        2. ngStyle: Adds/removes inline styles.

  23. What is Dependency Injection ?

  24. Difference between Static Routing and Dynamic Routing ?
    Static routing is a process where we manually set routing of components to navigate different pages.For example:-We create three components like,home,login,singup components then we set all three routing in app-routing-module.ts,in this file routes variable is there to set the path of component-wise.After that in html file create html link with a routerLink to set the components links.
    Static routing means routes are predefined in the source code and do not change at runtime.
    Dynamic routing means routes are created or modified at runtime based on logic, user roles, or fetched data.
    Routes can change while the app is running.
    Useful for role-based access or lazy loading.
  25. What is Lazy loading and why we use it?
    Lazy loading is a technique that allows you to load modules and components only when they are needed,reducing the initial load time of an application. For example we have thousand of component in our project and thousand of links to render that project,when user visit in our website then all component loaded then our website get slow to resolve that problem we use lazy loading,lazy loading only load that page that we need other's are not loaded,like user click on service then only service page load not all pages load then our website rendering fast.
  26. Difference between Observables and Subjects ?
    1. In Observable all subscriber have seperate execution.Observable does not allow user to manually push the data.
    2. In Subject all subscriber get the same execution. Subject allow user to manually push the data.
